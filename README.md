# work-smart: 智能工作规范 Skill

[![Work Smart](https://img.shields.io/badge/Work_Smart-高效工作-blue?style=for-the-badge)](https://github.com/vdlsz596/work-smart-skill)

> **先确认，再动手。先诊断，再修改。最小改动，不重写。**

`work-smart` 是一个为 Claude 设计的智能工作规范 Skill，确保 AI 代理在执行任务时遵循最佳实践，避免常见错误，提高工作效率和安全性。

## 🌟 核心功能

### 四阶段工作流

1. **需求确认** - 避免在未明确目标和约束时盲目编码
2. **调研** - 先查本地再查外部，只提取结论
3. **决策方案** - 提供多个选项并评估优缺点
4. **执行** - 强制备份、精准修改、结果验证

### 微改动快速通道

针对明确声明的微小改动（如"改错别字"、"修变量名"），自动跳过方案决策环节：

- **触发词**: `改错别字`/`修单词`/`改一行`/`改变量名`/`单字符修正`等
- **简化流程**: 诊断 → 极简确认 → 执行
- **安全保留**: 依然强制备份，遇到意外问题自动退回完整流程

```
确认改为 [目标]？回复 1 / y 继续
```

### 与 Plan Mode 协同

- 已通过 `/plan` 完成设计阶段的任务：自动跳过需求确认、调研和决策阶段
- 其他任务：按完整四阶段流程执行

### 与 caveman 压缩模式兼容

- 日常响应启用简洁模式
- 安全关键节点自动恢复完整表达（操作确认、备份提醒、不可逆操作警告）

## ✨ 为什么需要 work-smart？

避免 AI 编码助手常见错误：
- 没做需求确认就直接动手
- 调研阶段大段复述原文
- 修改前不做备份
- 一上来就准备重写全部
- 出错后不告知用户就自己试方案

`work-smart` 确保：
- 每次改动前都有清晰目标
- 每次修改都有备份保障
- 每次执行都经过用户确认
- 每个决策都有合理选项

## 📄 使用示例

### 标准任务
```
用户：帮我添加用户登录功能

1. **需求确认**  
   - 目标：实现安全的用户登录系统  
   - 约束：需支持第三方登录（Google/GitHub）  
   - 交付物：前端组件+后端API端点

2. **调研**  
   - 检查现有认证模块结构  
   - 分析会话管理机制

3. **决策方案**  
   - 方案A：JWT令牌认证（优点：无状态；缺点：需处理令牌刷新）  
   - 方案B：会话Cookie（优点：简单；缺点：需服务端存储）  
   → 用户选择方案A

4. **执行**  
   - 备份相关文件  
   - 添加新API路由和验证逻辑  
   - 验证：成功登录并获取用户信息
```

### 微改动
```
用户：改个错别字，把"recieve"改为"receive"
→ 系统进入快速通道，一步确认后完成修复
```
